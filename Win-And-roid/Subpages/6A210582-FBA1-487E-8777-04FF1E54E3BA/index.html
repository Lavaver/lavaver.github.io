为你的 Windows 10/11 设备安装 Windows Subsystem For Android

本教程将指导你安装 Windows Subsystem For Android ，并帮助你快速上手。

先决条件

  • 你的电脑 CPU 支持虚拟化，且打开了 Hyper-V 。若你不知道如何打开 Hyper-V ，可点击此处跳转到相应部分
  • 知道一些基本的 ADB 命令（例如安装/卸载应用程序等）。如果不知道也不要慌，你可以去搜索一些 ADB 工具箱来帮助你顺利调教好环境
  • 电脑必须安装了 Windows 10/11 ，且 UWP 功能没有被阉割（即其他人封装的系统不能移除与 UWP 相关的功能）
  • 准备一个剩余空间 ≥64GiB 的磁盘（尽量使用 SSD）
  • 电脑内存至少准备 8G 及以上

开始安装

  1. 从下载服务中下载后缀名为 .Msixbundle 的软件包，并保存在熟知的路径

  1. 双击下载好的软件包，准备安装

  1. 点按“安装”，并按照提示给予管理员权限后自动开始部署安装。这时候你可以坐和放宽，喝喝咖啡休息一下

  1. 如果看到了应用界面，那么就说明安装成功了！

如何打开虚拟化

你的设备安装的 Windows 版本必须至少为专业版及以上。家庭版或 S 模式不能打开

  1. 在搜索栏搜索“控制面板”，并打开它

  1. 单击“程序”类别，然后单击“启用或关闭 Windows 功能”

  1. 在打开的新窗口中勾选 Hyper-V

如果你的电脑 BIOS 没有直接的虚拟化开关，则你需要一并勾选“Windows 虚拟机监控程序平台”和“虚拟机平台”

  1. 单击“确定”，然后按要求指引重启整机即可开启 Hyper-V

使用 ADB 连接到 WSA

  1. 从此处下载 ADB ，并解压到熟悉的路径
  2. 在解压后地址栏输入 cmd ，进入命令提示符
  3. 此时你应该确保 WSA 的开发者模式和 ADB 调试已开启。当你开启调试后，WSA 会显示一个内网 IP 和端口
  4. 此时，你可以通过 adb connect <IP>:<Port> 命令来连接设备。如果一切顺利，你会看到 connected to <IP>:<Port> 字样

尝试使用 ADB 安装喜爱的应用程序

  1. 在 ADB 存储路径上方的地址栏输入 cmd ，进入命令提示符
  2. 如果你未连接 WSA ，你可以按照上面的简易教程快速连接。如已连接，你可进行下一步
  3. 在终端输入 adb install <软件路径> 并回车，ADB 会自动开始安装软件。当一切顺利时，你会在终端看到 Success 字样
